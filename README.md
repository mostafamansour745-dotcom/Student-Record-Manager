# Student Record Manager

A console-based student management system written in C# that demonstrates Object-Oriented Programming (OOP) principles and basic operations.

---

## Key Features

* **View Available Courses:** Browse through all the offered academic courses.
* **Register for a Course:** Enroll in a course and add it to the student profile.
* **View My Courses:** Display the list of courses currently registered by the student.
* **Exit:** Safely terminate the application.

---

## OOP Principles and Concepts Applied

### 1. Core OOP Pillars
* **Abstract Class:** Used as a blueprint for generalized entities that cannot be instantiated directly.
* **Inheritance:** Implemented to allow derived classes to reuse and extend code from base classes.
* **Polymorphism:** Achieved via method overriding to provide specific behavior dynamically at runtime.
* **Interface:** Applied to define strict contracts and achieve decoupled, pluggable code design.
* **Encapsulation:** Enforced by hiding internal data using private fields and exposing them safely via public properties.

### 2. Advanced C# Mechanics
* **Enum:** Used to represent fixed sets of named constants (e.g., Course types, Student levels).
* **Static:** Applied for utility functions and shared global state across the application.
* **Operator Overloading:** Customized standard operators to simplify object-to-object comparisons and operations.
* **Indexer:** Implemented to allow object instances to be indexed closely like arrays (e.g., managing student lists).

### 3. Data Structures and Control Flow
* **Exception Handling:** Robust try-catch blocks to gracefully handle runtime errors and validate user input.
* **Composition:** Designing classes that contain instances of other classes to represent "has-a" relationships.
* **List:** Utilized dynamic List collections for flexible memory management and data manipulation.
